MeetSync Menu Bar App

Product Requirements Document (PRD) for a Native, API-Accelerated macOS Transcription Utility

System Architecture

Designing a frictionless bridge between native macOS workflows, local file systems, and ultra-fast cloud LLM APIs.

The Menu Bar Experience

The application will reside entirely within the macOS Menu Bar, providing a completely unobtrusive user experience. It acts as a silent background daemon until user interaction is required.

  • Zero Dock Clutter: Runs silently without occupying Dock space.
  • Instant Status: The icon dynamically updates to reflect current processing states.
  • Quick Access: A single click reveals recent transcripts, current status, and quick-access to preferences.

UI & State Management

Configuration UI (Preferences)

Clicking "Preferences" opens a native SwiftUI window. This serves as the control center for the app.

Required Fields:
1. Groq API Key (Secure text field)
2. OpenRouter API Key (Secure text field)
3. Input Directory (Folder picker for Audio Hijack output)
4. Output Directory (Folder picker for Markdown summaries)

Dynamic Visual States

The Menu Bar icon must communicate the system's state silently to the user.

Idle: Standard monochromatic icon.

Processing: Subtle pulse or spin indicating Groq/OpenRouter API activity.

Error: Red indicator. Clicking reveals specific error (e.g., "Invalid API Key" or "Network Offline").

The Automation Workflow

1. The Trigger

User hits Cmd+Shift+R. A macOS Shortcut asks for Meeting Name, passes it to Audio Hijack, and starts recording.

2. Native Audio

Audio Hijack records app audio natively. User switches AirPods and uses volume keys without any friction.

3. Folder Watcher

Meeting ends. Audio Hijack saves the .mp3. MeetSync detects the new file instantly via background monitoring.

4. API Execution

MeetSync sends .mp3 to Groq (Whisper). Text is forwarded to OpenRouter (Claude). Summary saved to disk.

Technical Requirements

  • Native SwiftUI Development: The application must be built using Swift/SwiftUI to ensure maximum battery efficiency and native performance on the M5 architecture. Avoid cross-platform wrappers (Electron/Tauri) to minimize memory overhead.
  • Efficient File Monitoring: Utilize native macOS DispatchSourceFileSystemObject or FSEvents for zero-overhead folder watching. Avoid interval-based polling to preserve battery life.
  • Asynchronous API Handling: Implement `URLSession` with async/await for calls to Groq and OpenRouter. Ensure the main thread is never blocked, keeping the Menu Bar UI responsive during 60-second processing windows.
  • Audio Independence: The app relies strictly on Audio Hijack for file generation. It must not attempt to install virtual audio drivers, ensuring the user's Bluetooth and volume hardware controls remain entirely uninterrupted.

Ready for Development

This architecture decouples heavy audio routing from the automation logic, providing a robust, premium experience.
